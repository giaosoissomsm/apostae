  🚀 COMO COMEÇAR
═══════════════════════════════════════════════════════════════════════════

1. EXTRAIR ARQUIVO
   unzip apostae-v4.zip
   cd apostae

2. INSTALAR DEPENDÊNCIAS
   npm install

3. CONFIGURAR AMBIENTE
   cp .env.example .env
   # Editar .env com credenciais PostgreSQL + Redis

4. CRIAR BANCO DE DADOS
   # Ver MIGRATION_GUIDE_V4.md para instruções
   createdb apostae
   psql apostae -U postgres -c "CREATE USER apostae_user WITH PASSWORD 'senha';"

5. EXECUTAR MIGRATIONS
   npm run migrate

6. INICIAR SERVIDOR
   npm start

Acesse: http://localhost:3000

  🎓 O QUE MUDOU
═══════════════════════════════════════════════════════════════════════════

v3.0 → v4.0 ALTERAÇÕES MAIORES:

[DATABASE]
  SQLite                       → PostgreSQL
  Sessions em SQLite           → Redis
  Cache em memória             → Redis
  
[ARQUITETURA]
  Rotas monolíticas            → Controllers → Services → Repositories
  Middleware básico            → 4 middlewares especializados
  Config hardcoded             → env.js parametrizável
  
[SEGURANÇA]
  Validação simples            → Rate limiting + Bcrypt + Audit logs
  Sem IDOR check               → Validação em todas as rotas
  SQL concatenado              → Prepared statements 100%
  
[FUNCIONALIDADES]
  Usuários: 0 filtros          → Paginação + busca + filtros
  Wallet: N/A                  → Sistema completo
  Branding: Zoeira Market      → ApostaE

═══════════════════════════════════════════════════════════════════════════
  🆘 TROUBLESHOOTING RÁPIDO
═══════════════════════════════════════════════════════════════════════════

PostgreSQL connection refused:
  → Ver MIGRATION_GUIDE_V4.md seção PostgreSQL

Redis connection refused:
  → Verificar se Redis está rodando (redis-cli ping)
  → Ver MIGRATION_GUIDE_V4.md seção Redis

Migrações falhando:
  → npm run migrate rollback
  → npm run migrate
  → Ver migration.log para detalhes

Usuários antigos não aparecem:
  → npm run seed
  → Importar dados de apostas.db antigo
