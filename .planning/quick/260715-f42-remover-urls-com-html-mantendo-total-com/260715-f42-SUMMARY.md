# Phase quick-260715-f42 Plan 01: Remove .html URLs, keep clean extensionless routes Summary

Removed `.html` extensions from all public page URLs in ApostaE (server-side redirects + all
internal navigation references), while correcting the pre-existing bug where `/admin` served the
dashboard shell instead of the actual admin panel.

## What Was Built

**Task 1 тАФ server.js clean-URL routing:**
- Added a `/\.html$/i` route registered *above* `express.static`, mapping each legacy `*.html`
  path (`LEGACY_TO_CLEAN`) to its clean equivalent and issuing a `301` redirect with the query
  string preserved verbatim (`req.originalUrl.slice(req.path.length)`). Unmapped `.html` paths
  fall through via `next()` to the existing JSON 404 handler.
- Changed `express.static` to pass `{ index: false }` so it never auto-serves the index file at
  `/`.
- Replaced the old vestigial SPA route (`app.get(['/', '/dashboard', '/admin', '/profile'], ...)`
  which incorrectly sent `index.html` for `/admin`) with an explicit `CLEAN_TO_FILE` map and a
  loop registering one `app.get(route, ...)` per entry, each `res.sendFile`-ing the correct page
  file: `/` тЖТ index, `/login` and `/cadastro` тЖТ login, `/admin` тЖТ admin, `/profile` тЖТ profile,
  `/password-expires` тЖТ password-expires. `/dashboard` was dropped (never a real page).
- No catch-all route added; no new npm dependency.

**Task 2 тАФ HTML hrefs and canonical tags:**
- Updated 7 nav `href` attributes across `index.html`, `admin.html`, `profile.html` to point at
  clean extensionless paths.
- Updated both `login.html` inline-script post-auth redirects
  (`location.href = '/index.html'` тЖТ `/`) тАФ one for the already-logged-in bounce, one for the
  post-login/register redirect.
- Added a `<link rel="canonical">` tag to all five page files, each pointing at its own clean
  URL (login's canonical is `/login`, not `/cadastro`).

**Task 3 тАФ JS navigation and page-detection guards:**
- `public/js/api.js`: updated the 401 guard to exact-match (`location.pathname !== '/login' &&
  location.pathname !== '/password-expires'`) instead of `.endsWith('login.html')` /
  `.endsWith('password-expires.html')`; updated all five redirect targets
  (401 login redirect, 403 password-expires redirect, `requireLogin`, `requireAdminPage`
  non-admin bounce, `logout`) to clean paths.
- `public/js/session-timeout.js`: updated the early-return guard to exact-match
  (`location.pathname === '/login' || location.pathname === '/password-expires'`) instead of
  `.includes(...)` against the old `.html` substrings.
- `public/js/password-expires.js`: updated the index redirect and login redirect targets to
  clean paths.
- `src/middleware/auth.js`: updated `isPasswordChangeRoute`'s literal from
  `/password-expires.html` to `/password-expires` (this branch is dead code since `requireAuth`
  is only mounted on `/api/*` routers тАФ a no-op consistency fix so no stale `.html` path name
  remains in the tree).

## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 3 - Blocking] Duplicate process manager caused EADDRINUSE on restart**
- **Found during:** Overall verification (starting the server to run live curl checks)
- **Issue:** Both a `systemd` unit (`apostas.service`) and `pm2` (`apostas`, id 0) were configured
  to run `server.js` on port 3000. `pm2` was the one actually holding the port; `systemd`'s unit
  was enabled but had never been the active listener. Restarting via `systemctl restart` failed
  with `EADDRINUSE` because `pm2`'s process still held the port.
- **Fix:** Disabled and stopped the redundant `systemd` unit (`systemctl stop apostas.service &&
  systemctl disable apostas.service`), then restarted the actual running process via
  `pm2 restart apostas`. No application code was affected by this fix тАФ it only removed a
  duplicate, non-authoritative process manager. `pm2` is confirmed as the sole running instance
  post-fix.
- **Files modified:** None (system service configuration only, not part of `files_modified`).
- **Commit:** N/A (infrastructure/ops action, not a code change).

Otherwise: plan executed exactly as written тАФ no other deviations.
