<interface_context>
Event payload contract (produced by Plan 04's emitters, consumed by this plan's listeners — both plans MUST agree on these shapes). Per-user events carry `userId`; market-level events carry a recipient/refund array so listeners never re-query (Pitfall 4):

- `wager.placed`  → { wagerId, userId, marketId, question, choice, amount }
- `wager.cancelled` → { wagerId, userId, marketId, question, amount }
- `wager.won`     → { wagerId, userId, marketId, question, amount, payout }
- `wager.lost`    → { wagerId, userId, marketId, question, amount }
- `market.closed` → { marketId, question, recipients: [userId, ...] }
- `market.resolved` → { marketId, question, outcome, recipients: [userId, ...] }
- `market.deleted` → { marketId, question, refunds: [{ userId, wagerId, amount }, ...] }

Idempotency key mapping (type, related_entity, related_id):
- wager.* → (event name, 'wager', wagerId)
- market.closed / market.resolved → (event name, 'market', marketId)
- market.deleted → ('market.deleted', 'wager', wagerId)  // per-wager, so unique per affected user
</interface_context>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| domainEvents bus → notificationService listeners | Untrusted/duplicate event delivery must not spam or crash |
| authenticated caller → repository (by-id) | An id argument must never return/mutate another user's row |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-01-01 | Information Disclosure / Tampering | notificationRepository markRead/findById | high | mitigate | Every id-taking method also requires and applies `userId` in WHERE (Task 1); service returns NotFoundError (404) on non-owned id, no existence leak (Task 2) |
| T-01-02 | Denial of Service (inbox spam) | notificationService.notify() | medium | mitigate | UNIQUE constraint + local `23505` catch treats duplicate delivery as idempotent no-op (Task 2) |
| T-01-03 | Denial of Service (process crash) | notificationService listeners | high | mitigate | safeHandler try/catch on every listener; errors logged, never rethrown to the shared bus (Task 2) |
| T-01-04 | Denial of Service (resource exhaustion) | findByUserId limit param | medium | mitigate | Server-side `Math.min(limit,100)` cap regardless of client input (Task 1) |
</threat_model>
