<objective>
Wire the five existing financial call sites to emit the D-05 catalog events on `domainEvents`, completing the chain so real user/admin actions produce notifications: `placeWager` → `wager.placed`, `cancelWager` → `wager.cancelled` (wagerService); `closeMarket` → `market.closed`, `resolveMarket` → `market.resolved` + per-wager `wager.won`/`wager.lost`, `deleteMarket` → `market.deleted` (marketService). Purpose: Without this wiring the notification pipeline never fires from real actions. This is the producer half of the chokepoint; it must not alter any existing financial transaction boundary (PROJECT.md financial-integrity constraint).
Output: edited wagerService.js and marketService.js; new emission test.
Backend only (D-07).
</objective>

Emit these exact payload shapes (must match Plan 02's listener contract):
- wager.placed  → { wagerId, userId, marketId, question, choice, amount }
- wager.cancelled → { wagerId, userId, marketId, question, amount }
- wager.won     → { wagerId, userId, marketId, question, amount, payout }
