<objective>
Fix `marketService.resolveMarket()`'s win-branch payout to pay only the wager's *remaining* (post-cashout) stake, not the full original `potential_payout` — closing the double-pay bug RESEARCH.md Pitfall 2 identifies in already-shipped, security-reviewed financial code.

Purpose: Without this edit, a user could cash out 80% of a wager and still receive the full original payout on resolution (CASHOUT-03 requires the remaining stake to be "still eligible for resolution/payout on the non-cashed-out portion" — implying only that portion). This is a required edit to existing code, not new code, and is scoped as its own plan because it touches already-shipped, previously-reviewed logic and needs its own regression-focused verification.
Output: `resolveMarket()`'s payout calculation reduced-to-old-behavior when no cashout occurred; new resolution-integration tests proving both the regression-free case and the scaled case.
</objective>
